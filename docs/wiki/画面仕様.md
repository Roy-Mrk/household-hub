## ログイン画面 `/login`

### 機能

- **メール/パスワード ログイン**
- **メール/パスワード 新規登録**
  - 表示名（`display_name`）の入力欄を追加表示
  - 登録後に確認メールを送信
- **Google OAuth ログイン**
  - ボタンクリックで Google 認証へリダイレクト
  - `/auth/callback` でコードをセッションに交換

### 認証後

認証済みの場合はミドルウェアが `/` にリダイレクト。

## 支出管理 `/expense`

### 機能

| 機能 | 説明 |
|---|---|
| 一覧表示 | 50 件/ページのページネーション |
| 検索 | 説明（source）の部分一致検索 |
| 日付絞り込み | 開始日・終了日フィルター |
| 新規追加 | 「追加」ボタンで `EditEntryModal` を開く |
| 編集 | 行クリックまたは編集ボタンで `EditEntryModal` を開く |
| 削除 | 削除ボタン → 確認ダイアログ → 削除実行 |
| カテゴリ選択 | `CategorySelect` で階層選択 |

### 入力項目（EditEntryModal）

| フィールド | 入力形式 | バリデーション |
|---|---|---|
| 説明（source） | テキスト | 1〜100 文字 |
| 金額（amount） | 数値 | 0 以上の整数 |
| 日付（entry_date） | 日付ピッカー | 有効な日付 |
| カテゴリ | CategorySelect | 任意 |
| 区分（owner） | トグル: 自分/共有 | — |
| 精算対象（needs_settlement） | チェックボックス | 共有のみ表示 |

## 世帯管理 `/household`

### 状態 A: 世帯未所属

- 世帯名入力フォーム
- 「世帯を作成」ボタン → API `POST /api/households`
- 作成後は状態 B に遷移

### 状態 B: 世帯所属済み

| 機能 | 説明 |
|---|---|
| 世帯情報表示 | 世帯名・作成日 |
| メンバー一覧 | ロール（オーナー/メンバー）・参加日 |
| 招待リンク生成 | 「招待リンクを生成」ボタン → 有効期限 72h のトークン発行 |
| 招待リンクコピー | クリップボードにコピー |
| 世帯退出 | メンバーのみ可能。確認ダイアログ後に実行 |
| 世帯解散 | オーナーのみ可能。確認ダイアログ後に全データ削除 |

## 精算 `/settlement`

### 機能フロー

```
1. 精算対象エントリの選択
   └─ needs_settlement=true の共有支出・収入を一覧表示
   └─ チェックボックスで対象を選択

2. 割り勘比率の設定
   └─ メンバーごとに比率（%）を入力
   └─ 合計が 100% になるようにリアルタイム検証

3. 精算内容の確認
   └─ 各メンバーの負担額（公平な取り分）を表示
   └─ 支払フロー（誰が誰にいくら払うか）を表示

4. 確定ボタン → POST /api/settlement
   └─ 対象エントリの needs_settlement を false に更新
   └─ 精算レコードを作成
```

### 支払フロー計算アルゴリズム（settlementCalc.ts）

1. 各メンバーの **実際の支払額** を集計（支出 − 収入）
2. 比率から **公平な負担額** を計算
3. **差分（超過/不足）** を算出
4. 貪欲法で最小の送金回数になるよう支払フローを生成

## 精算履歴 `/settlement/history`

### 機能

- 精算レコードの一覧（ページネーション）
- 各レコードの表示: 精算日・合計金額・ステータス（有効/キャンセル済み）
- 詳細ページへのリンク

---

## 精算詳細 `/settlement/history/[id]`

### 表示内容

| セクション | 内容 |
|---|---|
| ヘッダー | 精算日時・合計金額 |
| 割り勘比率 | メンバーアバター付きの比率表示 |
| 支払フロー | 誰が誰にいくら支払うか |
| 対象エントリ一覧 | 含まれる収入・支出の詳細 |
| キャンセルボタン | 精算を取り消し（needs_settlement を true に戻す） |

---

## 招待受諾 `/invite/[token]`

### フロー

```
1. トークンの検証
   ├─ 期限切れ → エラーメッセージ表示
   ├─ 使用済み → エラーメッセージ表示
   └─ 有効 → 世帯名をプレビュー表示

2. 「参加する」ボタン → POST /api/households/join

3. 参加完了 → /household へリダイレクト
```

---

## 設定 `/settings`

### 機能

- **表示名変更**
  - 現在の表示名を表示
  - 新しい表示名を入力して保存
  - 同じ世帯内での重複チェック（`409 Conflict` の場合はエラー表示）
