## データフロー

```
ブラウザ（Client Component）
  │
  │  fetch('/api/...', { cache: 'no-store' })
  ▼
Next.js API Route  app/api/*/route.ts
  │  ① Supabase Auth でセッション検証
  │  ② Zod スキーマでリクエスト検証
  │  ③ supabaseAdmin（service role）で DB 操作
  ▼
Supabase PostgreSQL
```

- 全ページは `'use client'` のクライアントコンポーネント。`useState` / `useEffect` でローカル状態管理。
- API ルートは `supabaseAdmin`（サービスロールキー）を使用し、RLS をバイパスして直接 DB にアクセスする。認可ロジックはアプリケーション層で実装。
- グローバル状態管理ライブラリは使用しない。

## 認証フロー

```
1. 未認証ユーザーがアクセス
        ↓
   middleware.ts が /login へリダイレクト

2. /login でログイン方法を選択
   ├─ メール/パスワード
   │      → 新規登録：確認メール送信 → メール認証後ログイン
   │      → ログイン：セッション発行
   └─ Google OAuth
          → /auth/callback でコードをセッションに交換

3. セッションは httpOnly Cookie に保存
   （@supabase/ssr が自動管理）

4. 初回ログイン時に public.profiles レコードを自動作成
   （display_name は auth メタデータまたはメールのローカルパートから取得）

5. 認証済みユーザーが /login にアクセス → / にリダイレクト
```

## パスエイリアス

`tsconfig.json` で以下のパスエイリアスを設定しています。

```
@/*  →  ./src/*
```

使用例:

```typescript

import { CreateSchema } from '@/lib/validation';
```

## 主要コンポーネント

### EditEntryModal.tsx

収入・支出エントリの作成/編集に使用する共通モーダル。

| Props | 型 | 説明 |
|---|---|---|
| `open` | boolean | 表示/非表示 |
| `type` | `'income'\|'expense'` | 収入/支出の種別 |
| `initial` | object\|null | 編集対象データ（null で新規） |
| `onSave` | async function | 保存コールバック |
| `error` / `setError` | string / function | エラー状態 |

- `owner`（自分/共有）の最後の選択を `localStorage` に保持
- 共有エントリのみ精算フラグを表示

### CategorySelect.tsx

カテゴリ・サブカテゴリを選択するドロップダウン。
- 左列：カテゴリ一覧（★ = マスターデータ）
- 右列：サブカテゴリ一覧 + インライン追加フォーム
- 外側クリックで自動クローズ

### MonthlyBarChart.tsx

`/api/report/monthly` から取得した過去 12 ヶ月の収支を棒グラフで表示（Recharts）。

### CategoryPieChart.tsx

カテゴリ別の金額割合を円グラフで表示（Recharts）。5% 未満のスライスはラベル非表示。
